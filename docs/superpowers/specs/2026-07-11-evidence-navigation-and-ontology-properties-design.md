# 图谱证据定位与本体类属性展开设计

日期：2026-07-11

状态：已确认设计，待实施计划

## 背景与目标

当前图谱页面已经显示关系摘要、关系证据、属性和属性证据，但这些内容是静态列表。用户点击关系或属性后，无法直接定位到对应原文。本体页面的类型卡片也只展示类型基本信息，没有展开该类可用属性。

本次改造目标：

1. 点击画布关系线/标签或右侧关系摘要，定位到对应关系证据。
2. 点击实体属性行，定位到对应属性证据。
3. 点击本体类卡片，展示该类自己的属性和继承属性。
4. 保持现有项目切换、实体详情、审核入口和图谱性能行为不变。

## 非目标

- 不改变实体、关系、属性断言和证据的数据模型。
- 不改变证据内容或字符偏移的计算方式。
- 不在本次改造中增加证据编辑功能。
- 不默认展开所有本体属性，避免页面信息过载。

## 交互设计

### 关系点击

画布上的关系线/标签和右侧“关系摘要”都可点击。点击后：

1. 设置 `selectedRelationId`；
2. 画布关系线加粗并使用强调色；
3. 右侧关系摘要对应条目显示选中样式；
4. 请求关系证据接口；
5. 右侧滚动到对应关系证据卡片；
6. 目标证据卡片短暂高亮。

关系 ID 使用现有图谱边的 `id`，与 Neo4j Fact/RELATED 关系 ID 对齐。二度关系即使不属于当前中心实体详情，也必须能独立查询证据。

### 属性点击

实体详情的属性行改为可点击。点击后：

1. 设置 `selectedAttributeId`；
2. 属性行显示选中样式；
3. 右侧滚动到该属性的第一条证据；
4. 对应属性证据卡片短暂高亮。

多条属性证据只定位到第一条，但其余证据继续显示。

### 本体类点击

本体页的类型卡片可展开/收起属性定义。展开内容显示：

- 属性名称和 ID；
- 值类型；
- 是否允许多值；
- 枚举值（如果有）；
- 属性说明；
- 属性来源：本类定义或父类继承。

类型卡片默认收起，避免一次展示全部属性。子类同时显示自己的属性和 `effective_property_definitions` 中的继承属性。

## 后端接口与数据流

新增接口：

```text
GET /api/graph/relations/{relation_id}?project_id=<project>
```

响应复用现有 `RelatedFact`/`EvidenceDetail` 结构，至少包含：

- 关系 ID和类型；
- source、target 实体 ID和名称；
- 审核状态；
- 全部关联原文证据。

接口必须：

- 限制 `project_id`，禁止跨项目读取；
- 过滤被拒绝关系和已合并实体；
- 关系不存在时返回 404；
- 证据缺失时返回关系但证据为空，不生成虚假证据。

属性证据继续复用现有实体详情接口返回的数据，不新增属性查询接口。本体属性定义继续由 `/api/ontology` 返回的 `property_definitions` 和 `effective_property_definitions` 提供。

## 前端状态与组件边界

`GraphPage` 管理：

```text
selectedRelationId
selectedAttributeId
relationEvidence
relationEvidenceLoading
```

`GraphCanvas`：

- 接收 `onSelectEdge`；
- 根据 `selectedRelationId` 设置关系线样式；
- 不负责加载证据。

`EntityPanel`：

- 接收关系和属性选中回调；
- 为关系摘要、属性行和证据卡片提供稳定 DOM ID；
- 执行滚动和短暂高亮；
- 保留现有审核按钮。

`OntologyPage`：

- 管理展开的类型 ID；
- 渲染属性定义表；
- 不自行推导属性，直接使用 API 返回的有效属性定义。

切换项目或关闭实体详情时清理关系、属性选中状态和关系证据，避免残留旧项目内容。

## 视觉反馈

- 选中关系线：宽度增加，颜色使用主题强调色；
- 选中关系摘要：左侧强调线和浅色背景；
- 选中属性行：浅色背景和强调边框；
- 定位到的证据卡片：短暂高亮后恢复普通样式；
- 本体属性区域：与现有类型卡片风格一致，使用表格或紧凑列表；
- 保持移动端右侧面板的内部滚动，不改变页面整体布局。

## 错误处理

- 关系证据请求失败：保留关系选中状态，显示可重试的错误提示，不清空当前实体详情；
- 关系不存在或跨项目：显示“关系证据不可用”；
- 属性没有证据：属性仍可显示，但定位按钮不可用并标注“暂无证据”；
- 项目切换期间的旧请求必须取消或忽略，防止旧证据覆盖新项目。

## 测试与验收标准

### 后端测试

- 关系证据接口返回关系和全部证据；
- 不存在关系返回 404；
- 跨项目关系不可读取；
- 被拒绝关系和已合并实体被过滤；
- 无证据关系返回空证据列表。

### 前端测试

- 点击画布关系线触发关系证据加载；
- 点击关系摘要滚动到正确关系证据；
- 点击属性行滚动到正确属性证据；
- 关系/属性证据卡片高亮状态正确；
- 项目切换清理定位状态；
- 点击本体类型卡片展开属性定义；
- 子类显示继承属性和值类型信息。

### 验收标准

1. 点击截图中的 `KNOWS` 关系后，右侧定位到该关系原文证据。
2. 点击“身份/性别/称号”等属性后，定位到对应属性原文。
3. 点击“人物/门派/组织”等本体类后，显示该类可用属性及继承关系。
4. 二度关系、无证据关系和跨项目请求均按设计处理。
5. 现有图谱搜索、实体详情、审核和项目切换测试保持通过。
