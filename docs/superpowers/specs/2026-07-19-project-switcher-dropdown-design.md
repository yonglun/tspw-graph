# 项目选择器视口避让设计

## 背景

页面头部当前使用原生 `select` 展示项目。原生选项弹层由浏览器和操作系统定位，应用 CSS 无法可靠控制展开方向；当浏览器窗口靠近屏幕顶部时，弹层可能越过可视区域。

## 方案比较

1. **自定义无障碍 Listbox（采用）**：完全控制定位、尺寸与交互，可以稳定约束在视口内；需要补齐键盘和焦点行为。
2. 保留原生 `select` 并移动头部：改动小，但不能保证不同浏览器的弹层位置，且会破坏现有头部布局。
3. 引入第三方浮层组件库：定位能力完整，但仅为一个下拉框增加依赖，超出当前需求。

## 交互设计

- 收起状态保持现有“当前项目 + 项目名称”视觉结构。
- 点击触发按钮后显示自定义列表；列表通过 portal 挂载到 `document.body`，避免被头部容器裁切。
- 根据触发按钮的视口坐标计算弹层位置：优先向下展开；下方不足时向上展开；无论方向如何，弹层始终保留至少 8px 视口边距。
- 列表最大高度为视口可用空间与 320px 的较小值，超出内容在列表内部滚动。
- 当前项目显示选中标记；选择项目后复用现有 `setProjectId` 数据流并关闭列表。
- 项目切换被构建任务锁定时，触发按钮禁用，并保留现有说明文案。
- 点击外部、按 `Escape`、页面路由变化或窗口尺寸变化时关闭列表。

## 无障碍与键盘

- 触发器使用按钮语义，暴露 `aria-haspopup="listbox"`、`aria-expanded` 和关联列表 ID。
- 列表使用 `role="listbox"`，项目使用 `role="option"` 与 `aria-selected`。
- 打开时焦点定位到当前项目；支持 `ArrowUp`、`ArrowDown`、`Home`、`End`、`Enter`、`Space` 和 `Escape`。
- 关闭后焦点返回触发按钮。
- 保留可通过 `getByLabelText('当前项目')` 或等效可访问名称定位控件的测试契约。

## 组件边界

- `ProjectSwitcher` 继续负责读取项目上下文和提交项目切换。
- 新增一个专用的视口安全下拉组件，只负责展开状态、定位、键盘导航和选项呈现，不感知项目业务。
- 定位逻辑封装为可单元测试的纯函数，输入触发器矩形、视口尺寸和预期高度，输出方向、顶部、左侧、宽度及最大高度。

## 验收标准

- 在窗口位于屏幕顶部、普通桌面宽度和窄屏下，项目列表均不超出浏览器可视区域。
- 项目数量较多时列表内部滚动，页面本身不因展开产生横向滚动。
- 鼠标和完整键盘操作均可选择项目；焦点行为符合预期。
- 构建期间仍不能切换项目。
- 现有项目切换、URL 同步和跨页面项目保持行为不回归。

## 测试

- 定位纯函数覆盖向下、向上、窄屏右侧避让和最大高度限制。
- 组件测试覆盖打开/关闭、选择、禁用、外部点击和键盘导航。
- 更新现有 `ProjectContext`、`App`、图谱页与问答页测试，使其通过用户可见交互选择项目。
- 浏览器验证桌面端与窄屏，确认弹层边界和滚动行为。
