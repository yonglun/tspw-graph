# 构建任务真实进度与操作锁定设计

## 1. 背景

构建工作台目前存在两个相互关联的问题：

1. 上传接口返回任务后，上传表单、属性补抽和项目切换会重新启用，用户可以在已有任务仍运行时发起其他操作。
2. 任务模型虽然包含 `completed_chunks` 和 `total_chunks`，但 Worker 从未在处理过程中更新这两个字段，因此界面始终显示 `0 / 待切分`，进度条也无法反映真实进度。

此外，当前耗时的模型抽取和图谱导入都集中在 `IMPORTING` 阶段执行，界面长期显示“写入 Neo4j”，与实际工作不符。现有暂停接口只更新数据库状态，无法中断已经进入内存流水线的 Worker，也没有持久化片段中间结果，不能安全地断点续跑。

## 2. 目标

本次改动实现以下目标：

- 构建期间显示真实的片段处理进度；
- 构建期间阻止会创建新任务或改变当前任务上下文的操作；
- 保留安全、可预期的取消能力；
- 修正误导性的阶段文案和进度反馈；
- 使用后端并发保护避免绕过前端重复创建任务；
- 不扩大为完整的 Worker 检查点和断点续跑重构。

## 3. 非目标

本次不实现：

- 片段结果持久化；
- Worker 崩溃后的片段级断点续跑；
- 安全暂停和继续；
- 将切分、抽取、消解、校验、导入重构为五套独立持久化处理器；
- 强制终止正在进行的模型 HTTP 请求；
- 在 Neo4j 批量写入过程中强制中断。

安全暂停需要持久化每个片段的抽取结果、尝试次数和检查点，应作为单独的 Worker 架构升级实施。

## 4. 已确认的交互范围

任务处于非终态时，构建页面进入统一的 `processing` 状态。

### 4.1 禁止的操作

- 编辑项目标题；
- 重新选择上传文件；
- 切换构建模型；
- 再次点击“开始构建”；
- 切换属性补抽模型；
- 点击“重新抽取属性”；
- 使用右上角项目切换器切换项目。

### 4.2 保留的操作

- 查看当前任务状态和进度；
- 点击“取消”；
- 使用顶部导航进入只读页面。

“暂停”按钮从构建任务界面移除。后端暂停接口可以暂时保留以避免破坏兼容性，但界面不再声明当前版本支持安全暂停。

处理中，锁定区域显示说明：

> 当前任务处理中，完成或取消后可再次操作。

禁用控件需要同时具备原生 `disabled` 状态和清晰的视觉反馈，不能只依赖遮罩层或 CSS 阻止点击。

## 5. 真实进度模型

### 5.1 进度字段语义

- `total_chunks`：本次任务切分出的片段总数；
- `completed_chunks`：已结束处理的片段数，包括成功片段和因内容过滤被明确跳过的片段；
- `completed_chunks` 必须单调递增；
- `0 <= completed_chunks <= total_chunks`；
- 完成任务必须满足 `completed_chunks == total_chunks`。

因模型错误导致整个任务失败时，保留失败前的最后进度。取消任务时同样保留取消前的最后进度。

### 5.2 进度阶段

界面使用两种进度模式：

1. **不确定进度**
   - 文本尚未完成切分；
   - 所有片段处理完成后，正在归一化、汇总或写入 Neo4j。
2. **确定进度**
   - 已知 `total_chunks > 0` 且仍在逐片段处理；
   - 百分比按 `completed_chunks / total_chunks` 计算；
   - 同时显示“已完成 n / 总计 m 个片段”。

完成后固定显示 100%。失败或取消时保留最后的确定进度；如果失败发生在切分前，则继续显示无百分比的停止状态。

### 5.3 后端写回时机

Pipeline 切分文本后，通过进度回调写入：

```text
completed_chunks = 0
total_chunks = len(split.chunks)
```

每个片段完成以下任一结果后写回一次：

- 模型抽取和归一化成功；
- Azure 内容过滤被识别并作为可跳过片段记录。

其他不可恢复错误使任务失败，不虚增完成数。

每次进度写回都更新任务时间戳并生成 Job Event。现有 SSE 将事件推送到前端；SSE 不可用时，现有两秒轮询继续作为降级路径。

## 6. Worker 与取消语义

Pipeline 接收两个明确的协作接口：

- `on_progress(completed_chunks, total_chunks)`：持久化真实进度；
- `should_cancel()`：在片段边界检查任务是否已取消。

取消行为如下：

1. 用户点击“取消”，任务状态写为 `CANCELLED`；
2. 已经发出的模型请求不会被强制关闭；
3. 当前模型请求返回后，Pipeline 在下一个检查点停止；
4. 内存中尚未导入的结果被丢弃；
5. 不写入不完整图谱；
6. Worker 不得再把已取消任务覆盖为 `COMPLETED` 或 `FAILED`。

在最终 Neo4j 批量写入开始后，不允许取消。前端在该阶段隐藏或禁用取消按钮，后端也拒绝不合法的取消转换，以避免产生半写入状态。

当前任务在逐片段处理和最终导入时都使用持久化状态 `IMPORTING`，因此取消权限不能只根据状态名判断：

- `IMPORTING` 且 `completed_chunks < total_chunks` 时允许转为 `CANCELLED`；
- `IMPORTING` 且 `completed_chunks == total_chunks` 时视为已经进入最终汇总和导入，拒绝取消；
- `total_chunks == 0` 时允许取消尚未完成切分的任务。

Worker handler 返回后，Runner 必须重新读取任务状态；如果任务已经进入任何终态，不再执行旧快照对应的下一状态转换。

## 7. 任务阶段和文案

当前状态机暂不重构，但界面文案不再把整个耗时流水线描述成单纯的 Neo4j 写入：

- 切分前：`正在分析文本结构`；
- 已知片段总数并逐片处理：`抽取实体、关系与属性`；
- 片段全部完成后：`正在汇总并写入 Neo4j`；
- 完成：`构建完成`；
- 取消：`已取消`；
- 失败：`构建失败`。

由于当前 Pipeline 仍在 `IMPORTING` handler 内完成实际抽取，前端展示应优先根据进度字段区分子阶段，而不是直接照搬持久化状态名称。

## 8. 前端状态设计

`BuildPage` 根据当前 Job Snapshot 计算：

```text
processing = status 不属于 COMPLETED、FAILED、CANCELLED
```

该状态传给：

- `UploadStep`；
- `AttributeBackfill`；
- `ProjectSwitcher` 或项目上下文中的操作锁；
- `JobProgress`。

上传请求自身的 `busy` 状态仍保留，用于任务创建前的短暂请求阶段；控件最终禁用条件为 `busy || processing`。

项目切换锁只影响右上角选择器，不阻止顶部只读导航。锁定状态必须随 SSE 或轮询获得的终态快照立即解除。

`JobProgress` 使用原生 `<progress>` 的两种语义：

- 确定进度时设置 `value`；
- 不确定进度时不设置 `value`，并提供可见的动态样式。

所有进度文本放在 `aria-live="polite"` 区域，便于辅助技术获知更新，但避免每次动画帧产生播报。

## 9. 后端并发保护

前端禁用不能作为唯一保护。创建完整构建任务或属性补抽任务前，后端检查同一项目是否存在非终态任务：

- 存在时返回 HTTP `409`；
- 使用稳定错误码 `PROJECT_JOB_IN_PROGRESS`；
- 不创建第二条任务记录。

检查和创建必须位于同一数据库事务或采用等价的原子约束，避免两个并发请求同时通过检查。

对于新上传项目，项目与首个构建任务属于同一次创建流程，不受“已有同项目任务”误判影响。

## 10. 数据一致性与错误处理

- Repository 拒绝完成数倒退；
- Repository 拒绝完成数超过总数；
- 已进入终态的任务不接受后续进度写入；
- Worker 在设置下一状态前重新读取任务，终态任务不得被覆盖；
- SSE 推送失败不回滚任务进度；
- 取消检查发生在模型调用前和模型调用完成后；
- 内容过滤片段计入完成数，同时继续进入质量报告的失败片段和拒绝原因统计；
- 模型限流、超时和无效响应导致任务失败时保留最后成功进度。

## 11. 测试设计

### 11.1 后端单元与集成测试

- 切分后立即保存总片段数；
- 每个成功片段使完成数递增；
- 内容过滤片段也使完成数递增；
- 不可恢复错误不虚增当前片段；
- 完成任务达到 `n / n`；
- 失败任务保留最后进度；
- 取消后不再处理后续片段；
- 取消后 Worker 不覆盖终态；
- 最终导入阶段拒绝取消；
- Repository 拒绝进度倒退和越界；
- 同一项目存在活动任务时返回 `409 PROJECT_JOB_IN_PROGRESS`。

### 11.2 前端组件测试

- 非终态任务禁用上传表单全部字段和按钮；
- 非终态任务禁用属性补抽控件；
- 非终态任务禁用项目切换器；
- 顶部只读导航保持可用；
- 取消按钮在逐片段阶段可用；
- 最终导入阶段取消不可用；
- 未知总数时渲染不确定进度；
- 收到 SSE 进度快照后显示真实百分比和片段计数；
- SSE 降级轮询后仍可更新进度；
- 终态快照到达后解除操作锁；
- 暂停按钮不再显示。

### 11.3 浏览器与端到端验证

使用至少包含多个片段的测试文本运行构建任务：

1. 创建任务后确认所有变更型控件立即禁用；
2. 确认总片段数在切分后出现；
3. 观察完成数逐步递增，而不是只在最终完成时跳变；
4. 中途取消，确认后续片段停止且没有不完整图谱写入；
5. 完成任务，确认显示 100% 并恢复控件；
6. 断开 SSE，确认轮询仍能恢复最新进度；
7. 尝试绕过前端重复创建任务，确认后端返回 409。

## 12. 兼容与发布

现有 `jobs` 表已经包含两个进度字段，不需要数据库迁移。API Job Snapshot 结构保持兼容，只改变字段从长期为零变为真实值。

现有暂停 API 暂不删除，以免造成接口破坏；前端移除入口，并在后续 Worker 检查点项目中重新定义其可靠语义。

发布后正在运行的旧任务不会自动获得已完成片段的历史进度；新 Worker 版本处理的新任务将提供真实进度。部署应重建并重启 API、Worker 和 Web 容器。
